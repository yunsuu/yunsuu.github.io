### 함수형 프로그래밍 vs 객체형 vs 절차지향

[https://www.youtube.com/watch?v=4ezXhCuT2mw](https://www.youtube.com/watch?v=4ezXhCuT2mw)

- 함수형 프로그래밍이란 최근 ai, 블록체인, 빅데이터 등이 떠오르며 안전하게 병렬처리 하는게 중요해지며 부각되고 있다.
- 함수형 프로그래밍은 병렬처리에 안정성이 높는 장점
- 함수들을 적용하고 묶어서 프로그램을 구성해나가는걸 말다.
- opp에서 함수형 프로그래밍을 사용하는 이유
    1. 코드 가독성 → expression only
    2. 병렬처리 안전성 → 순수함수, 불변성
- 특징
    1. 순수함수
        - 함수에서 외부에서 상태값을 참조 or 변경하는걸 금지한다. 외부와의 영향을 일절 차단
    2. 비상태 불변성을 유지
        - 함수에 전달된 데이터를 변경하는게 아니라 새로운 오브젝트를 만들어서 사용
        - js에서는 object.freeze를 이용해 이를 가능케함
    3. expression only
        - if나 for 등을 사용하기보단 js기준으로 map같은 함수를 사용해서 코딩해야한다 → 남이 봤을때 가독성을 높이기 위해

    4. first-class, higer-order function이 가능해야한다.
        - first-class(일급함수) → 함수를 변수 or 인자에 할당 혹은 리턴이 가능해야함
        - higer-order(고차함수) → 함수에서 또 다른 함수를 리턴이 가능해야함

### 좋은코드는 어떤 코드라고 생각하며 이를 짜기 위해서는 어떻게 해야할까?

- 읽기 좋은 코드라고 생각하며 읽기 좋은 코드를 만들어 가는 방법은 정말 다양하다고 생각한다. 비유를 하자면 정원을 가꾸는 것과 비슷하다고 비유하고 싶다.
- 읽기 좋은 코드를 만드는 방법
    - 좋은 코드리뷰를 작성하는 것
    - 읽는 사람이 편하게 짜는 것 → 클린코딩 이론
    - 최근에는 함수형 프로그래밍을 공부하고 있는데 이것도 읽기 좋은 코드를 만들어가는 방법중 하나라고 생각
    - 그밖에 수많은 방법들이 있다고 생각
- 결국 읽기 좋은 코드를 만드는 방법은 특정 분야를 깊게 공부하기보단 코드에 애정을 담아 바라보기 좋은 코드를 짜기위한 방법이 하나둘씩 보이기 시작하는 것 같다.
- 취업준비가 끝나면 작가들이 필사하며 좋은 글들을 몸에 익히듯이 오픈소스 코드들을 읽어보면서 내 코드를 어떻게 해야 좀더 좋은코드로 만들 수 있을지 고민하고 공부해가는 활동을 하는게 계획이다.

### unit test, e2e test에 대한 설명 및 각 test의 중요성

- unit test는 코드 레벨에서의 테스트 e2e test는 사용자 레벨에서의 테스트
- unit test에서는 사용자에게는 느껴지지 않더라도 코드상에서의 문제점들을 빠르게 케치해내 후에 나올 수 있는 큰 에러들을 사전에 막을 수 있는 역활을 하고
- e2e test는 당장 유저들이 직접적으로 느낄 수 있는 서비스 장애를 손쉽게 찾아낼 수 있다는 장점이 있다
- unit test에서 중요한점은 각 복잡하게 얽혀있는 변수 혹은 state의 흐름을 정확히 파악하고 이를 바탕으로 mocking을 하는게 가장 중요하고
- e2e test에서는 실제 유저가 행동할 수 있는 예상치 못한 행동도 test에 반영하려고 하는 노력이 중요하다고 생각한다

### e2e test + unit test 방법 개선한거 구체적으로

- unit test는 코드안의 변수나 state의 구조를 파악해야하기 때문에 e2e test보다 작성시간이 더 많이 소요된다. 당시 상황은 서비스의 안정성보단 빠르게 개발해 유저에게 아이디어를 검증하는게 최우선순위 였기 때문에

### promise

- js에서는 비동기처리를 콜백으로 처리한다 이 방법의 문제점은 코드의 가독성을 해친다는 단점이 있다 이 단점을 극복하기 위해 promise가 사용된다.
- resolve, reject로 promise의 성공 실패 여부에 따라 로직따른 제어할 수 있다.
- 여러 비동기 작업들의 완료 여부를 확인하고 싶을때는 Promise.all을 사용한다
- promise를 쓰더라도 가독성의 문제가 완전히 해결되는게 아니라 async await 기술이 나왔다
- async await 에서는 try/catch를 이용해 에러 핸들링을 좀더 쉽고 보기 편하게 처리가 가능하다
- promise 체인으로 코드를 짤 경우 에러가 어느 체인에서 발생했는지 알려주진 않지만 async await은 이를 알 수가 있다.

### Node js 장점 및 단점 도입하기 좋은 상황

- 장점
    - Non-blocking I/O와 단일 스레드 이벤트 루프를 통한 높은 처리 성능
    - 이벤트 기반 비동기방식이라 서버 무리가 적다.
    - npm(node package manager)을 통한 다양한 모듈(패키지) 제공
- 단점
    - 이벤트 기반 비동기방식이라 서버단 로직이 복잡한 경우 콜백함수의 늪에 빠질 수 있다
    - 단일 쓰레드(Single Thread)이기 때문에 하나의 작업 자체가 많이 걸리는 웹서비스에는 어울리지 않다. 게시판형태와 같이 가벼운 I/O가 많은 웹서비스에 어울린다.

### **Node.js 가 어울리는 웹서비스**

- 간단한 로직.
- 대용량(동시에 여러 request를 처리)
- 빠른 응답시간 요구
- 빠른 개발 요구
- 비동기방식에 어울리는 서비스(네트워크 스트리밍 서비스, 채팅 서비스 등)

→ 게임서버 관련해서 node js가 갖는 장점

### **Node.js 가 어울리지 않는 웹서비스**

- 단일 처리가 오래 걸리는 경우 : 싱글 쓰레드이기 때문
- 서버 체크로직이 많은 경우 : 비동기방식이기 때문에 CallBack Hell에 빠지지 않기 위해
- 업무 복잡도/난이도가 높은 경우 : 에러가 나면 서버가 죽기 때문에 코드 품질 중요

    3. 변경이 일어나면 처리방법을 구현해야한다.
- 컨트롤러
    - 모델과 뷰를 잇는 다리역할
    1. 모델이나 뷰에대해서 알고있어야 한다 → 그래야 컨트롤 가능하니까
    2. 모델이나 뷰의 변경을 모니터링 해야한다.

### 함수형 프로그래밍 vs 객체형 vs 절차지향

[https://www.youtube.com/watch?v=4ezXhCuT2mw](https://www.youtube.com/watch?v=4ezXhCuT2mw)

- 함수형 프로그래밍이란 최근 ai, 블록체인, 빅데이터 등이 떠오르며 안전하게 병렬처리 하는게 중요해지며 부각되고 있다.
- 함수형 프로그래밍은 병렬처리에 안정성이 높는 장점
- 함수들을 적용하고 묶어서 프로그램을 구성해나가는걸 말다.
- opp에서 함수형 프로그래밍을 사용하는 이유
    1. 코드 가독성 → expression only
    2. 병렬처리 안전성 → 순수함수, 불변성
- 특징
    1. 순수함수
        - 함수에서 외부에서 상태값을 참조 or 변경하는걸 금지한다. 외부와의 영향을 일절 차단
    2. 비상태 불변성을 유지
        - 함수에 전달된 데이터를 변경하는게 아니라 새로운 오브젝트를 만들어서 사용
        - js에서는 object.freeze를 이용해 이를 가능케함
    3. expression only
        - if나 for 등을 사용하기보단 js기준으로 map같은 함수를 사용해서 코딩해야한다 → 남이 봤을때 가독성을 높이기 위해

    4. first-class, higer-order function이 가능해야한다.
        - first-class(일급함수) → 함수를 변수 or 인자에 할당 혹은 리턴이 가능해야함
        - higer-order(고차함수) → 함수에서 또 다른 함수를 리턴이 가능해야함
